## 3. Нововведения по категориям

### 3.1. 🔥 Quick Wins (1-2 недели)

#### 3.1.1. Интерактивный чек-лист
- **Суть:** Сделать чекбоксы в `checklist.html` интерактивными — клик → галочка → сохранение в `localStorage`.
- **Зачем:** Родители смогут отслеживать выполнение правил цифровой гигиены.
- **Сложность:** ⭐ (просто) — JavaScript + localStorage.

#### 3.1.2. CMS для блога
- **Суть:** Добавить таб "Блог" в `admin.html` с CRUD для статей (заголовок, текст, картинка, дата, категория).
- **Зачем:** Сейчас статьи захардкожены в HTML. Невозможно добавлять новый контент без программиста.
- **Сложность:** ⭐⭐ (средне) — новая таблица в БД, API endpoints, рендеринг на фронте.

#### 3.1.3. Живые ссылки Telegram и VK
- **Суть:** Заменить `#` на реальные URL групп/каналов.
- **Зачем:** Улучшает доверие и даёт канал связи.
- **Сложность:** ⭐ (тривиально).

#### 3.1.4. Исправление countdown
- **Суть:** Вместо хардкода `2026-05-25` — получать дату следующей главы из API или показывать "Скоро!" если дата прошла.
- **Зачем:** Сейчас показывает `00:00:00:00` — выглядит багом.
- **Сложность:** ⭐ (просто).

#### 3.1.5. Достижения / Бейджи
- **Суть:** Система достижений за прогресс:
  - "Первый шаг" — пройти 1 кадр
  - "Охотник за пикселями" — пройти runner
  - "Гимнаст глаз" — пройти gym 3 раза
  - "Мигалка" — пройти blink
  - "Трекер" — пройти tracker
  - "Глазастик" — пройти 1 главу
  - "Библиофил" — пройти все главы
  - "Богач" — накопить 500 монет
- **Зачем:** Геймификация, мотивация, shareable моменты.
- **Сложность:** ⭐⭐ (средне) — новая таблица achievements, логика проверки, UI бейджей.

#### 3.1.6. Ежедневная награда (Daily Reward)
- **Суть:** Заходишь каждый день → получаешь монеты. Streak (последовательность дней) даёт бонус. Пропуск дня = сброс streak.
- **Зачем:** Retention — дети будут возвращаться.
- **Сложность:** ⭐⭐ (средне) — логика streak, UI календаря.

### 3.2. ⚡ Medium (2-6 недель)

#### 3.2.1. Персонализация персонажа
- **Суть:** Дети могут кастомизировать Суперглазку — цвет костюма, маски, аксессуары (плащи, ботинки). Покупать за монеты.
- **Зачем:** Вовлечённость, смысл монет (сейчас они почти бесполезны), эмоциональная привязка.
- **Сложность:** ⭐⭐⭐ (сложно) — нужны SVG/PNG слои персонажа, UI кастомизации, интеграция с профилем.

#### 3.2.2. Ежедневные задания (Daily Quests)
- **Суть:** Каждый день 3 случайных задания:
  - "Пройди трекер"
  - "Прочитай 2 кадра"
  - "Сыграй в runner и набери 2 звезды"
- **Зачем:** Разнообразие, мотивация пробовать разные игры.
- **Сложность:** ⭐⭐⭐ (сложно) — система квестов, ротация, прогресс.

#### 3.2.3. Progressive Difficulty в играх
- **Суть:** Игры адаптируют сложность:
  - Runner: скорость растёт, появляются новые препятствия
  - Blink: меньше времени, больше кликов
  - Gym: босс имеет больше HP, цели движутся быстрее
  - Tracker: фигуры сложнее (восьмёрки, зигзаги)
- **Зачем:** Удержание интереса, тренировка нарастающей сложности.
- **Сложность:** ⭐⭐⭐ (сложно) — баланс, параметризация уровней.

#### 3.2.4. Родительский дашборд
- **Суть:** Отдельная страница `/parents/dashboard` (или раздел в `parents.html`):
  - Сколько времени ребёнок провёл в приложении
  - Какие игры и главы прошёл
  - Прогресс по достижениям
  - Рекомендации ("Ребёнок давно не играл в гимнастику")
  - Уведомления о новых главах
- **Зачем:** Родители хотят видеть пользу от приложения.
- **Сложность:** ⭐⭐⭐ (сложно) — сбор analytics, API для родителей, UI.

#### 3.2.5. Таймер 20-20-20
- **Суть:** Встроенный таймер, который напоминает каждые 20 минут: "Посмотри на что-то далёкое на 20 секунд". Можно включить в настройках.
- **Зачем:** Прямая польза для здоровья глаз, соответствует медицинской рекомендации.
- **Сложность:** ⭐⭐ (средне) — Web Notifications / внутриигровой popup, таймер.

#### 3.2.6. Аналитика использования
- **Суть:** Трекать время в приложении, какие кадры/игры чаще, где бросают. Показывать в профиле:
  - "Ты провёл 5 часов защищая глаза"
  - "Любимая игра: Runner"
  - "Прочитано кадров: 42"
- **Зачем:** Пользователь видит прогресс, родители видят пользу.
- **Сложность:** ⭐⭐ (средне) — localStorage/DB events, агрегация, UI.

#### 3.2.7. Leaderboard (таблица лидеров)
- **Суть:** Топ игроков по монетам, пройденным главам, streak. Можно сделать глобальный и среди друзей.
- **Зачем:** Соревновательный элемент, мотивация.
- **Сложность:** ⭐⭐⭐ (сложно) — API для лидеров, анонимизация детей (только никнеймы), защита от читерства.

### 3.3. 🚀 Long Term (1-3 месяца)

#### 3.3.1. Blink detection через камеру
- **Суть:** Использовать webcam + ML (MediaPipe Face Mesh или готовую библиотеку) для отслеживания частоты моргания. Если ребёнок долго не моргает — мягкое напоминание.
- **Зачем:** Уникальная фича, реальная польза для здоровья глаз.
- **Сложность:** ⭐⭐⭐⭐⭐ (очень сложно) — ML, приватность (всё на устройстве), производительность.

#### 3.3.2. Face-to-screen distance detection
- **Суть:** Через камеру определять расстояние лица до экрана. Если слишком близко — напоминание.
- **Зачем:** Профилактика близорукости.
- **Сложность:** ⭐⭐⭐⭐ (очень сложно) — требует ML, калибровку.

#### 3.3.3. Новые игровые механики
- **Суть:**
  - **Color Match** — игра на цветовое различение (тренирует дихоптическое зрение)
  - **Focus Shift** — переключение фокуса с близкого на дальнее (аналог 20-20-20)
  - **Pursuit** — следование за движущимся объектом (улучшает сглаживающие движения глаз)
  - **Saccades** — быстрое перемещение взгляда между точками
- **Зачем:** Расширение vision therapy, научная обоснованность.
- **Сложность:** ⭐⭐⭐⭐ (сложно) — новые игры с Canvas/WebGL.

#### 3.3.4. Сезонные ивенты
- **Суть:** Ограниченные по времени события:
  - "Новогодняя битва с Ленивусом" — зимний скин босса
  - "День зрения" — двойные монеты
  - "Летние каникулы" — новая глава бесплатно
- **Зачем:** Retention, вовлечённость, новостной повод.
- **Сложность:** ⭐⭐⭐ (сложно) — система ивентов, таймеры, специальный контент.

#### 3.3.5. Мультиязычность (i18n)
- **Суть:** Перевод на английский, казахский, украинский. Важно для масштабирования.
- **Зачем:** Рынок не ограничен русскоязычным сегментом.
- **Сложность:** ⭐⭐⭐ (сложно) — JSON-файлы переводов, замена всех строк, RTL (если нужно).

#### 3.3.6. Мобильное приложение
- **Суть:** Обёртка через Capacitor или PWA в TWA (Trusted Web Activity) для Google Play.
- **Зачем:** Push-уведомления, иконка на домашнем экране, родители находят в сторах.
- **Сложность:** ⭐⭐⭐ (сложно) — Capacitor build, нативные плагины, публикация.

### 3.4. 🛠️ Технические улучшения

#### 3.4.1. Analytics
- **Суть:** Внедрить Yandex Metrica и/или Google Analytics 4. События:
  - Старт главы / кадра
  - Завершение игры
  - Регистрация / логин
  - Покупка (если будет монетизация)
- **Зачем:** Понимание воронки, где пользователи уходят.
- **Сложность:** ⭐⭐ (средне).

#### 3.4.2. Sentry для ошибок
- **Суть:** Интегрировать Sentry.io для отслеживания JS-ошибок в production. Заменить/дополнить `error-reporter.js`.
- **Зачем:** Знать о багах до пользователей.
- **Сложность:** ⭐ (просто) — подключение SDK.

#### 3.4.3. Rate limiting
- **Суть:** Включить существующий `authLimiter` на auth routes.
- **Зачем:** Защита от brute-force.
- **Сложность:** ⭐ (просто).

#### 3.4.4. Guest → User миграция
- **Суть:** Endpoint для переноса прогресса и монет с guest token на зарегистрированный аккаунт.
- **Зачем:** Дети начинают играть как гости, родители потом создают аккаунт — прогресс должен сохраниться.
- **Сложность:** ⭐⭐ (средне).

#### 3.4.5. CI/CD Pipeline
- **Суть:** GitHub Actions: lint → test → build → deploy на сервер по SSH.
- **Зачем:** Автоматизация деплоя, меньше человеческих ошибок.
- **Сложность:** ⭐⭐ (средне).

#### 3.4.6. Unit тесты
- **Суть:** Тесты для MoodDetector, game logic, auth flows (Jest + jsdom).
- **Зачем:** Надёжность, рефакторинг без страха.
- **Сложность:** ⭐⭐⭐ (сложно) — настройка, покрытие.

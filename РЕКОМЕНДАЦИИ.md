## 🚨 Критично — стоит сделать в первую очередь

### 1. SEO и соцсети
| Проблема | Почему важно | Что сделать |
|----------|--------------|-------------|
| Нет Open Graph / Twitter Card | При шаринге в соцсети ссылка выглядит как «голый» URL | Добавить в `<head>`: `og:title`, `og:description`, `og:image`, `og:url`, `twitter:card` |
| Нет favicon | Во вкладке браузера — пустая иконка | Создать `favicon.ico` / `favicon.svg` и подключить |
| Нет `robots.txt` | Поисковики не знают, что индексировать | Создать `robots.txt` с `Allow: /` и указанием sitemap |
| Нет `sitemap.xml` | Поисковики медленнее находят страницы | Сгенерировать `sitemap.xml` со всеми страницами |
| Нет canonical URL | Возможны дубли страниц в поиске | Добавить `<link rel="canonical">` на каждую страницу |

### 2. PWA — сайт как «приложение»
- Создать `manifest.json` (название, иконки, цвета, display: standalone)
- Добавить Service Worker для **кеширования** — сайт будет работать оффлайн
- Добавить иконки в разных размерах (192×192, 512×512) для установки на телефон

### 3. Производительность
| Проблема | Решение |
|----------|---------|
| `superglazka.html` весит огромно из-за base64-изображений | Вынести картинки в отдельные файлы, использовать `loading="lazy"` |
| Шрифты загружаются без `font-display: swap` | Добавить `&display=swap` в Google Fonts URL |
| Нет предзагрузки критических ресурсов | Добавить `<link rel="preload">` для шрифтов и первого видео |
| Нет оптимизации изображений | Использовать WebP/AVIF вместо JPEG/PNG где возможно |

### 4. Доступность (Accessibility)
- Добавить `aria-label` на все кнопки без текста (иконки, аудио-кристалл, закрытие)
- Добавить `role="button"` и `tabindex="0"` для кастомных элементов
- Добавить `aria-live="polite"` для динамических текстов (субтитры, счётчик кадров)
- Проверить контрастность текста — некоторые светлые подписи на тёмном фоне могут не проходить WCAG AA

## ⚡ Функционал — улучшит вовлечённость

### 5. Система прогресса
- **Закладка «Продолжить»** — при возвращении на сайт предлагать начать с последнего кадра
- **Автосохранение** кадра в `localStorage` каждые 5 секунд
- **Индикатор прочитанных кадров** внутри эпизода (галочки на прогресс-баре)

### 6. Ачивки и мотивация
Сейчас есть только монеты. Добавить достижения:
- 🏆 «Первый кадр» — прочитал 1 кадр
- 🏆 «Глазастик» — прошёл все тренировки в эпизоде 1
- 🏆 «Марафонец» — прочитал 10 кадров подряд
- 🏆 «Быстрый бегун» — набрал 3 звезды в Runner
- 🏆 «Коллекционер» — собрал 1000 монет

### 7. Настройки пользователя
Модалка с настройками (шестерёнка в меню):
- Громкость озвучки (ползунок 0–100%)
- Громкость эффектов
- Скорость печати текста (медленно / нормально / быстро / мгновенно)
- Размер шрифта субтитров (маленький / средний / большой)
- Автовоспроизведение аудио (вкл/выкл)
- Уменьшение анимаций (для людей с вестибулярными нарушениями — `prefers-reduced-motion`)

### 8. Раздел «Для родителей»
Отдельная страница или секция на лендинге:
- Описание пользы проекта (развитие внимания, гигиена зрения)
- Рекомендации по времени использования
- Как объяснить ребёнку пользу упражнений
- Контакты для связи / обратная связь

### 9. FAQ / Помощь
Модалка с вопросами:
- «Как управлять?» (тапы, свайпы, пробел)
- «Что делать, если звук не работает?»
- «Как заработать монеты?»
- «Почему эпизод заблокирован?»

### 10. Контент-маркетинг
- **Блог / советы** — короткие статьи о здоровье глаз для детей (2–3 минуты чтения)
- **Чек-лист для родителей** — PDF «10 правил здоровья глаз ребёнка»
- **Видео-трейлер** — 30-секундное промо на лендинг
- **Галерея артов** — концепт-арты персонажей, обои на рабочий стол

## 🔒 Безопасность и юридика

### 14. Обязательно для детского проекта
- **Cookie-consent баннер** — GDPR / 152-ФЗ (даже если куки минимальные)
- **Политика конфиденциальности** — страница с текстом о сборе данных
- **Пользовательское соглашение** — особенно для формы регистрации (email родителя)
- **CSP (Content Security Policy)** — защита от XSS через `<meta http-equiv="Content-Security-Policy">`
- **Валидация email** на стороне клиента + сервера (если будет бэкенд)

## 📊 Аналитика и рост

### 15. Метрики
Подключить **бесплатную** аналитику:
- **Yandex.Metrika** или **Google Analytics 4** — базовые визиты
- **Яндекс.Вебмастер** / **Google Search Console** — индексация
- Кастомные события:
  - `episode_started` — какой эпизод запустили
  - `frame_viewed` — до какого кадра дошли
  - `game_started` / `game_completed` — какие игры популярны
  - `game_skipped` — где бросают
  - `registration_shown` / `registration_completed` — конверсия формы

### 16. Технический мониторинг
- Проверить консоль на ошибки — в `superglazka.html` есть огромные base64-строки, которые могут ломать парсинг
- Добавить `try/catch` вокруг `JSON.parse(localStorage)` — сейчас есть, но можно усилить
- Обработка ошибок загрузки видео/аудио — fallback на изображение + текст

## 🛠 Технический долг

### 17. Рефакторинг
| Проблема | Решение |
|----------|---------|
| Дублирование стилей между `style.css`, `modern.css`, `comic.css` | Объединить или разделить чётко по назначению |
| `superglazka.html` — монолит с inline JS/CSS | Вынести в отдельные файлы |
| Использование `innerHTML` в `app.js` | Заменить на `textContent` / создание DOM-элементов (XSS-защита) |
| Жёстко закодированные пути `assets/audio/ауд_1.mp3` | Вынести в конфиг-объект |
| Отсутствие TypeScript / JSDoc | Добавить JSDoc хотя бы к публичным функциям |

### 18. Тестирование
- Проверить на iOS Safari — есть известные баги с Web Audio и видео
- Проверить на слабом интернете (3G) — огромные base64 в `superglazka.html` не загрузятся быстро
- Проверить landscape-ориентацию на телефоне — сайт заточен под portrait
